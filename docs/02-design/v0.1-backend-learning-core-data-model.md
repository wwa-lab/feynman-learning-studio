# Data Model: v0.1 Backend Learning Core

**Source**: `architecture-to-design` skill native output  
**Primary Source**: `docs/01-specs/v0.1-backend-learning-core.md`

## Overview

v0.1 persists LearningTopic and LearningExperiment. LearningTopic organizes transformation areas; LearningExperiment represents runnable learning activities under a topic。

## Entity Relationship Diagram

```text
┌──────────────────────────────┐
│        LearningTopic          │
│  id PK                       │
│  slug UNIQUE                 │
│  name                        │
└───────────────┬──────────────┘
                │ 1
                │
                │ N
┌───────────────▼──────────────┐
│      LearningExperiment       │
│  id PK                       │
│  topicId FK                  │
│  title                       │
│  status                      │
└──────────────────────────────┘
```

## Entity Definitions

### LearningTopic

- **Table name**: `learning_topic`

| Column | Type | Nullable | Description |
|---|---|---:|---|
| `id` | Long | No | Primary key |
| `name` | String | No | topic name |
| `slug` | String | No | unique lowercase key |
| `description` | String | Yes | topic description |
| `active` | Boolean | No | whether topic is active |
| `created_at` | Timestamp | No | creation time |
| `updated_at` | Timestamp | No | update time |

Constraints:

- Primary key: `id`。
- Unique: `slug`。
- Slug format: lowercase letters, numbers, hyphens。

### LearningExperiment

- **Table name**: `learning_experiment`

| Column | Type | Nullable | Description |
|---|---|---:|---|
| `id` | Long | No | Primary key |
| `topic_id` | Long | No | FK to `learning_topic.id` |
| `title` | String | No | experiment title |
| `summary` | String | Yes | experiment summary |
| `difficulty` | String | No | difficulty enum |
| `status` | String | No | lifecycle status |
| `spec_path` | String | Yes | related spec path |
| `created_at` | Timestamp | No | creation time |
| `updated_at` | Timestamp | No | update time |

Constraints:

- Primary key: `id`。
- Foreign key: `topic_id` references `learning_topic.id`。
- `difficulty` must be `BEGINNER`, `INTERMEDIATE`, or `ADVANCED`。
- `status` must be a supported v0.1 status。

## State Models

### LearningExperiment Status

Valid states:

- `DRAFT`
- `READY`
- `RUNNING`
- `COMPLETED`
- `ARCHIVED`

Valid transitions:

```text
DRAFT -> READY
READY -> RUNNING
RUNNING -> COMPLETED
DRAFT -> ARCHIVED
READY -> ARCHIVED
RUNNING -> ARCHIVED
```
