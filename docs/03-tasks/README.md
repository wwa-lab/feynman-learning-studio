# Tasks

这个目录保存 implementation task lists、execution manifests 和 handoff notes。

新增 task list 时，从 `template.md` 开始。

建议使用方式：

```text
spec/design
  -> task list
  -> execution manifest
  -> implementation loop
  -> verification evidence
```

Manifest 的作用是减少猜测：明确输入、分支、scope、constraints、verification command 和风险。
