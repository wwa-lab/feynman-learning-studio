# v0.1 Backend Learning Core Runbook

## 目标

这个 runbook 用来复现 v0.1 backend learning core 的本地验证流程：

- 运行 Spring Boot backend。
- 使用 Flyway 管理 schema，并用 H2 in-memory database 完成本地测试。
- 打开 OpenAPI JSON 和 Swagger UI。
- 验证 Learning Topic 与 Learning Experiment 的核心 API。
- 记录当前阶段的已知限制与 Feynman 复盘问题。

如果你没有 Java 经验，建议先阅读 VS Code 手动验证指南：

```text
docs/04-runbooks/v0.1-vscode-manual-verification-guide.md
```

## 启动本地服务

在 `backend/` 目录执行：

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

默认端口：

```text
http://localhost:8080
```

本地服务使用 H2 in-memory database。schema 由 Flyway 在启动时迁移，服务重启后，示例数据会重置。

## Database Migration

本项目统一使用 Flyway 管理 database schema。

v0.1 初始 migration：

```text
backend/src/main/resources/db/migration/V1__create_learning_core_tables.sql
```

不要用临时 `schema.sql` 管理 application schema。后续 schema 变更应新增 migration，例如：

```text
V2__add_run_evidence_tables.sql
```

测试会检查 `flyway_schema_history`，证明 migration 已执行。

## 常见问题

### 8080 端口被占用

检查端口：

```bash
lsof -iTCP:8080 -sTCP:LISTEN -n -P
```

如果是上一次本地启动遗留的 Java 进程，可以停止该进程后重新运行。

### H2 与 PostgreSQL 差异

v0.1 默认使用 H2，后续 integration database 方向是 PostgreSQL。涉及 SQL function、index、constraint 语义时，需要在 PostgreSQL integration run 中再次验证。

### Swagger URL 选择

v0.1 文档使用：

```text
http://localhost:8080/swagger-ui/index.html
```

SpringDoc 默认入口 `/swagger-ui.html` 会跳转到 Swagger UI，本轮 smoke 已验证为 HTTP 302。

## 当前限制

- v0.1 没有 authentication 与 authorization。
- v0.1 使用 physical delete。
- v0.1 没有 frontend。
- v0.1 没有外部 AI provider integration。
- 本地 run 默认使用 H2 in-memory database，不保存重启后的数据。
