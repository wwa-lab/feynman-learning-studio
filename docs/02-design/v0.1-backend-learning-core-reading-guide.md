# v0.1 Backend Learning Core 中文阅读导览

## 目的

这份导览帮助你阅读 v0.1 的 design / task 文档。

由于本项目要求保留 skill-native output，部分文档标题和段落仍然会出现英文结构，例如 `Overview`、`Module Design`、`Acceptance`。这些英文不是为了增加理解成本，而是为了保留 skill 的原始输出边界。

阅读时可以按这份中文导览走。

## 推荐阅读顺序

### 1. 先读 Spec

文件：

```text
docs/01-specs/v0.1-backend-learning-core.md
```

你要回答：

- v0.1 要解决什么问题？
- 范围内做什么？
- 明确不做什么？
- Topic 和 Experiment 分别是什么？
- 怎么证明 v0.1 完成？

### 2. 再看 Architecture

Skill-native output：

```text
docs/02-design/v0.1-backend-learning-core-architecture.md
```

Project supplement：

```text
docs/02-design/v0.1-backend-learning-core-architecture-supplement.md
```

阅读重点：

- 这是一个 layered modular monolith。
- API layer 负责 HTTP boundary。
- Domain/service layer 负责 business rules。
- Persistence layer 通过 MyBatis 访问 database。
- Local evidence layer 用 tests、local run、Swagger/API examples 证明结果。

IBM iSeries 类比：

- API layer 像外部调用入口。
- Service layer 像集中处理业务规则的程序段。
- Mapper/XML 像集中维护 SQL access 的位置。
- Runbook 和 test output 像可复现的 joblog / 操作证据。

### 3. 看 Detailed Design

Skill-native output：

```text
docs/02-design/v0.1-backend-learning-core-design.md
```

Project supplement：

```text
docs/02-design/v0.1-backend-learning-core-design-supplement.md
```

阅读重点：

- `Common API Foundation`：统一 `ApiResponse`、分页结构和 exception handling。
- `Learning Topic Module`：Topic CRUD、slug 唯一性、被引用时不能删除。
- `Learning Experiment Module`：Experiment CRUD、topic reference、status transition。
- `Persistence Module`：MyBatis mapper + Flyway migration。
- `OpenAPI Module`：Swagger UI 和 `/v3/api-docs`。

### 4. 看 Data Model

Skill-native output：

```text
docs/02-design/v0.1-backend-learning-core-data-model.md
```

Project supplement：

```text
docs/02-design/v0.1-backend-learning-core-data-model-supplement.md
```

阅读重点：

- `learning_topic` 是 learning area / transformation topic。
- `learning_experiment` 挂在一个 topic 下。
- `learning_experiment.topic_id` 引用 `learning_topic.id`。
- 删除被引用 Topic 会失败，返回 `TOPIC_IN_USE`。
- Schema 现在由 Flyway migration 管理。

### 5. 看 API Guide

Skill-native output：

```text
docs/02-design/contracts/v0.1-backend-learning-core-api-guide.md
```

Project supplement：

```text
docs/02-design/contracts/v0.1-backend-learning-core-api-guide-supplement.md
```

阅读重点：

- 所有 API 都在 `/api` 下。
- Topic endpoints 管理 `LearningTopic`。
- Experiment endpoints 管理 `LearningExperiment`。
- `PATCH /api/experiments/{id}/status` 专门处理 status transition。
- 所有 response 都使用统一 envelope：

```json
{
  "success": true,
  "data": {},
  "errorCode": null,
  "errorMessage": null
}
```

## 关键技术名词

- `Spec`：范围、目标、验收标准的 source of truth。
- `Skill-native output`：skill 原始产物，尽量保留。
- `Project supplement`：为了满足本 repo 的 Harness gates 额外补充的内容。
- `Harness gate`：证明结果符合预期的检查点。
- `Runbook`：如何运行、验证、排障和记录学习问题。
- `Flyway migration`：有版本的 database schema 变更脚本。
- `MyBatis mapper`：Java interface + XML SQL mapping 的 persistence 入口。

## 阅读时不用纠结的点

- 不需要一开始看懂所有 Java 细节。
- 不需要背 Spring Boot 注解。
- 不需要先理解所有 MyBatis XML。
- 先抓住 flow：HTTP request -> controller -> service -> mapper -> database。
- 再抓住 evidence：test output -> local run -> Swagger smoke -> runbook。
