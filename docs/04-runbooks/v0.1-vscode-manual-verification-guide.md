# v0.1 VS Code 手动验证指南

## 读者定位

这份指南面向有 IBM iSeries 经验、但没有 Java / Spring Boot 经验的学习者。

你可以先这样建立心智模型：

| Java / Spring Boot 世界 | 可以类比成 IBM iSeries 世界里的什么 |
| --- | --- |
| VS Code | 你的开发工作台 |
| JDK | Java 程序的运行环境，类似运行某类程序必须具备的 runtime |
| Maven | 构建、依赖下载、测试运行工具，类似把 compile、依赖管理、test command 统一起来 |
| Spring Boot application | 一个本地启动的 backend service job |
| H2 in-memory database | 临时测试库，服务重启后数据清空 |
| REST API | 外部程序通过 HTTP 调用的接口 |
| Swagger UI | API 的可视化测试面板 |
| `curl` | 命令行 API 调用工具，适合留下可复现证据 |

本轮不是要你立刻成为 Java 工程师，而是要你能完成一个完整的验证闭环：

```text
打开项目 -> 检查环境 -> 运行测试 -> 启动 backend -> 打开 Swagger -> 手动验证 API -> 记录问题
```

## 1. 打开项目

1. 打开 VS Code。
2. 选择 `File -> Open Folder...`。
3. 打开这个目录：

```text
/Users/leo/wwa-lab/GitHub/feynman-learning-studio
```

确认左侧 Explorer 能看到：

```text
backend/
docs/
AGENTS.md
```

如果你只打开了 `backend/`，也能跑 Java，但看不到完整 SDD 文档链路。学习时建议打开 repository root。

## 2. 安装 VS Code 扩展

在 VS Code 左侧 Extensions 面板搜索并安装：

- Extension Pack for Java
- Spring Boot Extension Pack
- Maven for Java
- REST Client，可选，用来在 VS Code 内发 HTTP 请求

说明：

- `Extension Pack for Java` 让 VS Code 能识别 Java project、跳转类、运行测试。
- `Spring Boot Extension Pack` 让 VS Code 更容易启动 Spring Boot application。
- `Maven for Java` 让你在 VS Code 里看到 Maven lifecycle。
- `REST Client` 不是必须；本指南默认用 Swagger UI 和 Terminal。

## 3. 检查 Java 与 Maven

打开 VS Code Terminal：

```text
Terminal -> New Terminal
```


## 4. 理解项目结构

本轮最重要的是 `backend/`：

```text
backend/
  pom.xml
  src/main/java/com/feynman/learningstudio/
  src/main/resources/application.yml
  src/main/resources/db/migration/
  src/test/java/com/feynman/learningstudio/
```

关键文件说明：

- `pom.xml`：Maven 项目配置，声明 Spring Boot、MyBatis、Druid、H2、SpringDoc 等 dependency。
- `application.yml`：本地服务配置，包含端口、H2 database、MyBatis mapper 路径。
- `db/migration/`：Flyway migration 目录，启动时按版本创建或更新 database schema。
- `src/main/java/.../topic`：Learning Topic 的 API、service、mapper、domain。
- `src/main/java/.../experiment`：Learning Experiment 的 API、service、mapper、domain。
- `src/test/java/...`：自动化测试。

如果用 IBM iSeries 的视角看：

- `db/migration/V1__...sql` 近似于有版本号的 DDL 变更脚本。
- `mapper XML` 近似于集中维护 SQL access。
- `service` 是业务规则层。
- `controller` 是对外暴露 API 的入口。

## 5. 先运行自动化测试

在 VS Code Terminal 执行：

```bash
cd backend
mvn test -Dspring.profiles.active=test
```

第一次运行 Maven 会下载 dependency，可能较慢。看到大量下载日志是正常的。

成功时你应该看到：

```text
Tests run: 28, Failures: 0, Errors: 0, Skipped: 0
BUILD SUCCESS
```

教练提示：

- 这一步相当于先跑一遍系统自检。
- 如果测试失败，不要先打开 Swagger。先把失败信息记下来，因为 backend 可能还没处在可验证状态。
- `target/` 是 Maven 生成目录，类似编译产物，不需要手动改，也不应该提交。

## 6. 启动 backend

继续在 `backend/` 目录执行：

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

成功时你会看到类似：

```text
Tomcat started on port 8080
Started FeynmanLearningStudioApplication
```

这表示 backend service 已经在本机启动：

```text
http://localhost:8080
```

注意：

- 这个 Terminal 要保持运行，不要关闭。
- 如果你按 `Ctrl + C`，服务会停止。
- H2 是 in-memory database，重启服务后刚才手动创建的数据会消失，这是 v0.1 的预期行为。

## 7. 打开 Swagger UI

打开浏览器，访问：

```text
http://localhost:8080/swagger-ui/index.html
```

你应该看到 Swagger UI 页面，里面有这些 API group：

```text
learning-topic-controller
learning-experiment-controller
```

如果打不开，先检查：

1. backend Terminal 是否还在运行。
2. 日志里是否有 `Tomcat started on port 8080`。
3. 是否有端口冲突。

检查端口：

```bash
lsof -iTCP:8080 -sTCP:LISTEN -n -P
```

## 8. 在 Swagger 创建 Topic

在 Swagger UI 中找到：

```text
POST /api/topics
```

点击：

```text
Try it out
```

填入 request body：

```json
{
  "name": "Spring Boot Backend Basics",
  "slug": "spring-boot-backend-basics",
  "description": "练习 Spring Boot backend CRUD 与本地验证。",
  "active": true
}
```

点击：

```text
Execute
```

成功时你应该看到：

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Spring Boot Backend Basics",
    "slug": "spring-boot-backend-basics",
    "description": "练习 Spring Boot backend CRUD 与本地验证。",
    "active": true
  },
  "errorCode": null,
  "errorMessage": null
}
```

记住这个 `id`。如果服务刚启动，通常是 `1`。

## 9. 在 Swagger 创建 Experiment

找到：

```text
POST /api/experiments
```

填入：

```json
{
  "topicId": 1,
  "title": "实现第一个 Spring Boot CRUD API",
  "summary": "用 TDD 实现 topic create API 并通过 H2 integration test 验证。",
  "difficulty": "BEGINNER",
  "status": "DRAFT",
  "specPath": "docs/01-specs/v0.1-backend-learning-core.md"
}
```

如果你的 Topic id 不是 `1`，把 `topicId` 改成刚才创建 Topic 返回的 `id`。

成功时你应该看到：

```json
{
  "success": true,
  "data": {
    "id": 1,
    "topicId": 1,
    "title": "实现第一个 Spring Boot CRUD API",
    "difficulty": "BEGINNER",
    "status": "DRAFT"
  },
  "errorCode": null,
  "errorMessage": null
}
```

## 10. 验证状态流转

找到：

```text
PATCH /api/experiments/{id}/status
```

把 path parameter `id` 填成刚才 Experiment 返回的 `id`。

request body：

```json
{
  "status": "READY"
}
```

成功时应该返回：

```text
success: true
status: READY
```

这个规则可以这样理解：

- `DRAFT`：草稿，还没准备好执行。
- `READY`：准备好了，可以开始执行。
- `RUNNING`：正在执行。
- `COMPLETED`：完成。
- `ARCHIVED`：归档。

v0.1 不允许随便乱跳状态。状态流转失败时，会返回 `INVALID_STATUS_TRANSITION`。

## 11. 验证一个业务保护规则

我们已经决定：

```text
不允许删除已经被 Experiment 引用的 Topic。
```

在 Swagger 找到：

```text
DELETE /api/topics/{id}
```

填入刚才 Topic 的 `id`，点击 Execute。

你应该看到：

```json
{
  "success": false,
  "data": null,
  "errorCode": "TOPIC_IN_USE",
  "errorMessage": "Topic is used by experiments."
}
```

HTTP status 应该是：

```text
409
```

教练提示：

这不是系统坏了，而是业务规则生效了。它类似于你不能删除一个仍然被其他业务记录引用的主数据，否则下游记录会失去上下文。

## 12. 验证一个唯一性规则

再次调用：

```text
POST /api/topics
```

使用同一个 `slug`：

```json
{
  "name": "Duplicate Topic",
  "slug": "spring-boot-backend-basics",
  "description": "重复 slug 验证。",
  "active": true
}
```

你应该看到：

```json
{
  "success": false,
  "data": null,
  "errorCode": "DUPLICATE_TOPIC_SLUG",
  "errorMessage": "Topic slug already exists."
}
```

HTTP status 应该是：

```text
409
```

这说明 database unique constraint 和 service error handling 都工作了。

## 13. 用 OpenAPI JSON 做低层验证

Swagger 是可视化界面。OpenAPI JSON 是机器可读的 API 描述。

打开：

```text
http://localhost:8080/v3/api-docs
```

你应该看到 JSON，其中包含：

```text
openapi
info
paths
components
```

如果 Swagger UI 像“交互式 API 面板”，OpenAPI JSON 就像这组 API 的接口说明书原始数据。

## 14. 停止服务

回到正在运行 backend 的 Terminal，按：

```text
Ctrl + C
```

再次访问 Swagger UI 应该打不开。这说明服务已经停止。

## 15. 手动验证完成标准

完成 v0.1 手动验证时，你应该能打勾：

- VS Code 能打开 repository root。
- `java -version` 能看到 Java 17 或更高。
- `mvn -version` 能正常输出。
- `mvn test -Dspring.profiles.active=test` 通过。
- `mvn spring-boot:run -Dspring-boot.run.profiles=local` 能启动服务。
- Swagger UI 能打开。
- 能创建 Topic。
- 能创建 Experiment。
- 能把 Experiment 从 `DRAFT` 更新到 `READY`。
- 删除被引用 Topic 返回 `TOPIC_IN_USE`。
- 重复 Topic slug 返回 `DUPLICATE_TOPIC_SLUG`。
- 能说清楚 H2 in-memory database 为什么重启后数据会丢。
