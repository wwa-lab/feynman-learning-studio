# v0.1 Backend Learning Core — API Implementation Guide

**Source**: `architecture-to-design` skill native output  
**Date**: 2026-06-09  
**Version**: v0.1 Draft  
**Base Path**: `/api`  
**Backend stack**: Spring Boot 3.x, Maven, MyBatis, Druid, H2, OpenAPI/Swagger  
**Auth model**: None for v0.1

## Overview

This guide defines the REST API contract for Learning Topic and Learning Experiment. All endpoints return the common API envelope。

## Authentication

No authentication in v0.1. The API is for local learning only。

## Error Response Format

```json
{
  "success": false,
  "data": null,
  "errorCode": "VALIDATION_ERROR",
  "errorMessage": "Request validation failed."
}
```

## Endpoint Reference

### Create Topic

```text
POST /api/topics
```

Request body:

| Field | Type | Required |
|---|---|---:|
| `name` | String | Yes |
| `slug` | String | Yes |
| `description` | String | No |
| `active` | Boolean | No |

Error cases:

- `VALIDATION_ERROR`
- `DUPLICATE_TOPIC_SLUG`

### Get Topic

```text
GET /api/topics/{id}
```

Error cases:

- `VALIDATION_ERROR`
- `TOPIC_NOT_FOUND`

### List Topics

```text
GET /api/topics?pageNo=1&pageSize=10
```

Response data uses paginated shape。

### Update Topic

```text
PUT /api/topics/{id}
```

Error cases:

- `VALIDATION_ERROR`
- `TOPIC_NOT_FOUND`
- `DUPLICATE_TOPIC_SLUG`

### Delete Topic

```text
DELETE /api/topics/{id}
```

Behavior:

- If topic is referenced by experiments, return `TOPIC_IN_USE`。
- Otherwise physical delete。

### Create Experiment

```text
POST /api/experiments
```

Request body:

| Field | Type | Required |
|---|---|---:|
| `topicId` | Long | Yes |
| `title` | String | Yes |
| `summary` | String | No |
| `difficulty` | String | Yes |
| `status` | String | No |
| `specPath` | String | No |

Error cases:

- `VALIDATION_ERROR`
- `INVALID_TOPIC_REFERENCE`
- `INVALID_EXPERIMENT_STATUS`

### Get Experiment

```text
GET /api/experiments/{id}
```

Error cases:

- `VALIDATION_ERROR`
- `EXPERIMENT_NOT_FOUND`

### List Experiments

```text
GET /api/experiments?pageNo=1&pageSize=10&topicId=1
```

Response data uses paginated shape。

### Update Experiment

```text
PUT /api/experiments/{id}
```

Error cases:

- `VALIDATION_ERROR`
- `EXPERIMENT_NOT_FOUND`
- `INVALID_TOPIC_REFERENCE`

### Update Experiment Status

```text
PATCH /api/experiments/{id}/status
```

Request body:

| Field | Type | Required |
|---|---|---:|
| `status` | String | Yes |

Error cases:

- `VALIDATION_ERROR`
- `EXPERIMENT_NOT_FOUND`
- `INVALID_EXPERIMENT_STATUS`
- `INVALID_STATUS_TRANSITION`

### Delete Experiment

```text
DELETE /api/experiments/{id}
```

Behavior:

- Physical delete。

## State Reference

```text
DRAFT -> READY -> RUNNING -> COMPLETED
DRAFT -> ARCHIVED
READY -> ARCHIVED
RUNNING -> ARCHIVED
```

## Concurrency

No optimistic locking in v0.1。
