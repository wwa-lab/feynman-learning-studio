# Project Supplement: v0.1 Backend Learning Core Data Model

**Supplements**: `docs/02-design/v0.1-backend-learning-core-data-model.md`  
**This file is not skill-native output.**

## 1. Project Decisions

- v0.1 uses physical delete。
- `LearningTopic` cannot be deleted when referenced by `LearningExperiment`。
- Referenced topic delete returns `TOPIC_IN_USE`。
- PostgreSQL is the future integration database direction after H2。

## 2. Suggested Length Bounds

These bounds are project supplements for implementation consistency:

| Field | Suggested Bound |
|---|---:|
| `LearningTopic.name` | 100 |
| `LearningTopic.slug` | 80 |
| `LearningTopic.description` | 1000 |
| `LearningExperiment.title` | 160 |
| `LearningExperiment.summary` | 1000 |
| `LearningExperiment.specPath` | 500 |

## 3. Suggested Indexes

- Unique index on `learning_topic.slug`。
- Index on `learning_experiment.topic_id`。
- Optional index for deterministic list sorting on `created_at, id`。

## 4. H2 / PostgreSQL Compatibility Notes

- Use explicit columns in SQL。
- Avoid database-specific SQL unless documented in runbook。
- Keep timestamp handling simple for v0.1。
