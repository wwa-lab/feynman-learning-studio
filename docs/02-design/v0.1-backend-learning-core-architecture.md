## Overview

- **Architecture Summary**: v0.1 Backend Learning Core 是一个 Spring Boot 3.x backend slice，用于管理 Learning Topic 和 Learning Experiment。系统通过 REST / JSON API 暴露 CRUD 和 status update 能力，通过 MyBatis 访问 relational database，并用 H2 支持本地 tests。
- **Design Objective**: 建立第一个可运行、可测试、可解释的 backend learning experiment core。
- **Architectural Style**: Layered modular monolith。

---

## Source Specification

- **Feature / System Name**: v0.1 Backend Learning Core
- **Scope Summary**: 实现 Learning Topic CRUD、Learning Experiment CRUD、Experiment status update、统一 API response envelope、统一 exception handling、H2 local tests、OpenAPI/Swagger 和 local runbook。

---

## Architectural Drivers

### Key Functional Drivers

- 管理 Learning Topic。
- 管理 Learning Experiment。
- 维护 Learning Experiment lifecycle status。
- 所有 API 返回统一 envelope。
- 通过 Swagger/OpenAPI inspect API。
- 通过 H2-backed tests 证明本地可运行。

### Key Non-Functional Drivers

- v0.1 必须本地可运行，不依赖 Docker 或外部 database。
- API boundary 必须 validation。
- SQL 必须通过 MyBatis parameterized SQL 执行。
- Error response 不得暴露内部实现细节。

### Constraints and Assumptions

- JDK 17。
- Spring Boot 3.x。
- Maven。
- MyBatis。
- Druid。
- H2 for tests。
- OpenAPI/Swagger。
- No frontend in v0.1。
- No auth in v0.1。
- Future integration database direction: PostgreSQL。

### Layer Summary

- **API Layer**: 接收 HTTP request，执行 request validation，返回统一 response envelope。
- **Common Foundation Layer**: 管理 API envelope、pagination、error code、exception mapping。
- **Domain Layer**: 执行 topic 和 experiment 的 business rules。
- **Persistence Layer**: 通过 MyBatis mapper 访问 database。
- **Local Evidence Layer**: 通过 H2 tests、local run、Swagger/API examples 支撑 verification。

## Data Architecture

### Conceptual Entities

| Entity | Description | Key Attributes |
|---|---|---|
| LearningTopic | 学习主题或 transformation topic。 | id, name, slug, description, active, createdAt, updatedAt |
| LearningExperiment | 挂在 topic 下的 runnable learning activity。 | id, topicId, title, summary, difficulty, status, specPath, createdAt, updatedAt |

### State / Status Models

```text
DRAFT -> READY -> RUNNING -> COMPLETED
DRAFT -> ARCHIVED
READY -> ARCHIVED
RUNNING -> ARCHIVED
```

### Persistence Responsibilities

- Learning Topic 和 Learning Experiment 必须持久化。
- Topic slug 必须 unique。
- Experiment 必须引用已存在 Topic。
- 被 Experiment 引用的 Topic 不允许删除。

## Risks / Tradeoffs

- v0.1 无 auth，只适用于本地学习。
- physical delete 简单，但后续如需要 learning history，需要设计 archive/soft delete。
- H2 与后续 PostgreSQL 可能存在 SQL behavior 差异。
