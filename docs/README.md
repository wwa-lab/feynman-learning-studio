# Documentation

This directory keeps project knowledge close to the code.

## Structure

- `00-project-rules/`: project rules, development standards, and workflow.
- `01-specs/`: product and feature specs. Start from `01-specs/template.md`.
- `02-design/`: architecture and design notes.
- `03-tasks/`: task lists and execution manifests. Start from `03-tasks/template.md`.
- `04-runbooks/`: local test, local run, and operational runbooks.
- `05-decisions/`: ADRs and important project decisions.
- `06-articles/`: article drafts and public learning notes.
- `07-prompts/`: reusable prompts for external tools and agent handoffs.
